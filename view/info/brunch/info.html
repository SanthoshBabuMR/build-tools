About Grunt
  • Brunch is a Build tool
  • Brunch is based on Convention Over Configuration
  • Brunch is fundamentally specialized and geared towards building assets, these files that get used in the end by your runtime platform, usually a web browser. It comes pre-equipped with a number of behaviors and features.
    1. Categorization of source files: JavaScript, Style sheets, Templates and 'miscellanea;'
    2. Smart concatenation of these files towards one or more target files;
    3. Maintenance of all relevant source maps;
    4. Minification of resulting files if we're in production mode;
    5. Watching of source files to update the build on the fly.
  • Brunch is a pipeline, which makes of streams
  • Brunch uses incremental builds.
  • Brunch emphasizes the concept of skeleton, providing a kickstarter by creating a few folders, pre-requiring dependencies in package.json and providing a default Brunch configuration
  • brunch-config.js (or) brunch-config.coffee is the file in which brunch configurations reside
  • Brunch comes with a built-in HTTP server, listening on 3333 port.
Sample package.json
                              
{
  "name": "name-of-the-application",
  "version": "0.0.0",
  "description": "description/purpose of the application",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "repository" : { 
    "type" : "git|svn",
    "url"  : "path-to-repo"
  },
   "scripts": {
    "start": "brunch watch --server",
    "test": "brunch test"
  },
  "author": "name-of-the-author",
  "license": "ISC",
  "dependencies": {
    "javascript-brunch": ">= 1.0 < 1.8",
    "css-brunch": ">= 1.0 < 1.8",
    "uglify-js-brunch": ">= 1.0 < 1.8",
    "clean-css-brunch": ">= 1.0 < 1.8",
    "auto-reload-brunch": ">= 1.0 < 1.8"
  }
}
Sample Gruntfile.js

Convention Over Configuration 

                              
exports.config =
  # See http://brunch.io/#documentation for docs.
  files:
    javascripts:
      joinTo: 'app.js'
    stylesheets:
      joinTo: 'app.css'
    templates:
      joinTo: 'app.js'
  server:
  	port: 62343
